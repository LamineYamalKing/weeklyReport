## 1. 快速开始

### 1.1 启动项目

```bash
# 安装依赖（首次使用）
npm install

# 启动开发服务器
npm run dev
```

启动后浏览器自动打开 `http://localhost:3000`，页面会自动跳转到工作日志页。

### 1.2 推荐的首次使用流程

1. 先在**标签管理**中创建几个常用标签（如「前端开发」「会议」「Code Review」等）
2. 回到**工作日志**创建第一条记录
3. 积累一周后，前往**周报生成**页面生成并导出周报

## 2. 工作日志

工作日志是核心功能，用于记录每天的工作内容。

### 2.1 新建日志

1. 点击页面右上角的 **+ 新建日志** 按钮
2. 填写表单：
   - **日期**（必填）：工作发生的日期
   - **标题**（必填）：简洁描述工作内容
   - **描述**（选填）：详细说明，支持多行文本
   - **预计时长**（选填）：计划花费的小时数
   - **实际时长**（选填）：实际花费的小时数
   - **标签**（选填）：点击标签按钮进行关联
3. 点击 **创建** 保存

### 2.2 编辑日志

1. 在日志列表中，点击对应条目右侧的 **编辑**（铅笔）图标
2. 修改内容后点击 **保存修改**
3. 表单会自动滚动到页面顶部

### 2.3 删除日志

1. 点击对应条目右侧的 **删除**（垃圾桶）图标
2. 确认删除操作

### 2.4 时间追踪（计时器）

每条日志旁边有一个 **播放** 图标按钮：

1. 点击 **▶** 开始计时
2. 计时器以浮动条形式显示在页面右下角，实时显示经过时间
3. 点击 **停止** 后，自动将计时时长累加到该日志的「实际时长」中
4. 同一时间只能对一个日志进行计时

### 2.5 筛选与查询

页面顶部的筛选栏支持：

- **日期范围**：选择开始日期和结束日期
- **标签筛选**：按标签过滤日志
- **快捷按钮**：
  - **今天**：显示今天的记录
  - **本周**：显示本周一至今的记录
  - **清除筛选**：恢复显示全部记录

### 2.6 列表信息

- 每条日志右侧显示 **按时**（绿色）或 **超时**（橙色）标签，基于预计时长与实际时长的对比
- 页面顶部显示当前筛选条件下的总条数和总时长

## 3. 标签管理

标签用于对工作日志进行分类管理。

### 3.1 新建标签

1. 点击 **+ 新建标签** 按钮
2. 输入标签名称
3. 选择标签颜色：
   - 从预设颜色中选择
   - 或点击颜色选择器自定义颜色
4. 点击 **创建** 保存

### 3.2 编辑标签

- 鼠标悬停在标签卡片上，点击 **编辑**（铅笔）图标
- 修改名称或颜色后保存

### 3.3 删除标签

- 鼠标悬停在标签卡片上，点击 **删除**（垃圾桶）图标
- 删除后，所有日志中关联该标签的关系也会被移除

### 3.4 常用标签建议

| 类别 | 示例标签 |
|------|----------|
| 开发 | 前端开发、后端开发、数据库、API 开发 |
| 沟通 | 会议、Code Review、需求讨论 |
| 管理 | 项目管理、文档编写、周报 |
| 学习 | 技术调研、培训、自学 |

## 4. 周报生成

根据指定日期范围，自动生成工作内容汇总。

### 4.1 生成周报

1. 选择日期范围：
   - 手动选择开始和结束日期，或
   - 点击 **本周一至今** 快速选择当前周，或
   - 点击 **上周** 快速选择上一整周
2. 点击 **生成周报**

### 4.2 查看周报

周报按日期分组展示：
- 每天一个分组，包含当日所有工作条目
- 每条显示标题、时长、标签和描述
- 每天有小计时长，底部显示总计

### 4.3 导出周报

支持三种导出方式：

| 格式 | 说明 | 文件扩展名 |
|------|------|-----------|
| Markdown | 结构化的 Markdown 格式，适合粘贴到文档工具 | `.md` |
| 文本 | 纯文本格式，适合邮件或聊天发送 | `.txt` |
| 打印 | 浏览器直接打印，支持保存为 PDF | — |

导出文件命名规则：`周报_开始日期_结束日期.扩展名`

## 5. 数据统计

可视化展示工作时长分布情况。

### 5.1 概览卡片

页面顶部显示三个关键指标：
- **累计工作时长**：所有日志的总实际时长
- **总记录数**：创建的日志总数
- **日均工作时长**：有记录天数的平均工作时长

### 5.2 每日统计

- 以水平柱状图展示最近 30 天的工作时长
- 每根柱子代表一天，长度对应工作时长
- 底部显示合计数据

### 5.3 每周统计

- 以水平柱状图展示最近 12 周的周工作时长
- 帮助了解工作强度的周度趋势

### 5.4 标签统计

- 按标签汇总工作时长，从高到低排序
- 柱状图颜色与标签颜色一致
- 帮助了解时间在各类型工作上的分布

## 6. 常见问题

### 数据存储在哪里？

所有数据存储在本地 SQLite 数据库文件中（`./database/weekly-report.sqlite`），不会上传到任何外部服务器。

### 数据可以备份吗？

可以直接复制 `./database/weekly-report.sqlite` 文件进行备份。恢复时将文件放回原位置即可。

### 可以多人使用吗？

当前设计为单人使用，数据存储在本地。如需多人使用，建议各自运行独立实例。

### 导出为 PDF

在周报页面生成周报后，点击 **打印** 按钮，在打印对话框中选择「另存为 PDF」即可。

### 修改了标签颜色，已生成的周报会变化吗？

会的。周报实时读取数据，修改标签颜色后重新生成周报即可看到更新。

### 数据库文件损坏了怎么办？

如果数据库文件出现问题，删除该文件后重启开发服务器，系统会自动重建空数据库。
