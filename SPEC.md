# 周报工具需求规格文档 (SPEC.md)

## 项目概述

构建一个个人使用的周报工具，帮助用户记录每日工作内容、追踪时间、自动生成周报并支持导出。

---

## 目标用户

- **主要用户**：个人用户（开发者本人）
- **使用场景**：个人工作日志记录与周报生成

---

## 核心功能

### 1. 记录工作内容
- 用户可以创建、编辑、删除每日工作日志
- 每条记录包含：日期、标题、详细描述、创建/更新时间
- 支持查看历史记录

### 2. 自动汇总周报
- 根据选定日期范围自动汇总工作内容
- 支持按周、自定义时间范围生成报告
- 智能聚合和整理日志内容

### 3. 时间追踪
- 为每项工作记录预计时长和实际时长
- 支持启动/停止计时器功能
- 统计每日/每周总工作时间

### 4. 分类/标签
- 支持创建自定义标签和分类
- 每条工作记录可关联多个标签
- 支持按标签筛选和统计

### 5. 导出功能
- 支持 Markdown 格式导出
- 支持 Word (.docx) 格式导出
- 支持 PDF 格式导出
- 导出内容可自定义包含/排除字段

## 数据库设计

### 表结构

#### work_logs（工作日志表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| date | DATE | 工作日期 |
| title | VARCHAR(200) | 工作标题 |
| description | TEXT | 详细描述 |
| estimated_hours | DECIMAL(4,2) | 预计时长（小时） |
| actual_hours | DECIMAL(4,2) | 实际时长（小时） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

#### tags（标签表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| name | VARCHAR(50) | 标签名称 |
| color | VARCHAR(7) | 标签颜色（HEX） |
| created_at | DATETIME | 创建时间 |

#### work_log_tags（日志-标签关联表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| work_log_id | INTEGER | 日志 ID（外键） |
| tag_id | INTEGER | 标签 ID（外键） |

## API 设计

### 工作日志 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/logs | 获取日志列表（支持分页、日期筛选、标签筛选） |
| GET | /api/logs/:id | 获取单条日志详情 |
| POST | /api/logs | 创建新日志 |
| PUT | /api/logs/:id | 更新日志 |
| DELETE | /api/logs/:id | 删除日志 |

### 标签 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/tags | 获取所有标签 |
| POST | /api/tags | 创建新标签 |
| PUT | /api/tags/:id | 更新标签 |
| DELETE | /api/tags/:id | 删除标签 |

### 周报 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/reports/weekly | 生成周报（指定日期范围） |

### 统计 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/stats/daily | 每日工作时长统计 |
| GET | /api/stats/weekly | 每周工作时长统计 |
| GET | /api/stats/tags | 按标签统计工作时长 |

## 用户故事

### US-01: 创建工作日志
作为一个用户，我想要创建新的工作日志，以便记录我今天完成的工作内容。

**验收标准：**
- 可以输入日期、标题、描述
- 可以设置预计时长
- 可以选择关联标签
- 创建成功后显示在日志列表中

### US-02: 查看日志列表
作为一个用户，我想要查看我的工作日志列表，以便回顾历史工作记录。

**验收标准：**
- 默认显示最近 30 天的日志
- 支持按日期范围筛选
- 支持按标签筛选
- 列表显示：日期、标题、时长、标签

### US-03: 编辑工作日志
作为一个用户，我想要编辑已有的工作日志，以便修正或补充工作内容。

**验收标准：**
- 点击日志条目进入编辑模式
- 可修改所有字段
- 保存后更新列表显示

### US-04: 删除工作日志
作为一个用户，我想要删除不需要的工作日志，以便保持数据整洁。

**验收标准：**
- 支持删除单条日志
- 删除前有确认提示
- 删除后从列表中移除

### US-05: 时间追踪
作为一个用户，我想要记录每项工作的实际时长，以便准确了解时间花费。

**验收标准：**
- 可手动输入实际时长
- 可启动计时器自动计时
- 计时器显示实时经过时间
- 停止计时后自动保存实际时长

### US-06: 创建标签
作为一个用户，我想要创建自定义标签，以便对工作进行分类。

**验收标准：**
- 可输入标签名称
- 可选择标签颜色
- 创建后可用于日志分类

### US-07: 生成周报
作为一个用户，我想要自动生成周报，以便快速汇总本周工作。

**验收标准：**
- 可选择日期范围
- 自动聚合选中范围内的日志
- 按日期分组显示
- 显示每项工作的时长小计
- 显示总工作时长

### US-08: 导出周报
作为一个用户，我想要导出周报为多种格式，以便分享或归档。

**验收标准：**
- 支持 Markdown 格式
- 支持 Word (.docx) 格式
- 支持 PDF 格式
- 导出文件命名包含日期范围

## 非功能性需求

### 性能要求
- 页面首次加载时间 < 3 秒
- API 响应时间 < 500ms
- 支持至少 1000 条日志记录

### 安全要求
- 仅本地访问，无外部网络请求
- 数据存储在本地，不上传云端

### 可维护性
- 代码结构清晰，职责分离
- 关键逻辑添加注释说明
- 遵循 ESLint 规范
